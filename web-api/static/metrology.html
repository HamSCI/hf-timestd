📐 How It Works
The transmission time equation and what D_clock means
▼

The Measurement Problem

D_clock is the offset between your local system clock and UTC(NIST):

D_clock = Tsystem − TUTC(NIST)

This is extracted from HF time signals by solving the transmission time equation:

Tarrival = Temission + τpropagation + Dclock

The Variables

Tarrival Measured tone arrival time (RTP timestamp from GPSDO-disciplined SDR)
Temission Known transmission time (top of minute, UTC)
τprop Ionospheric path delay (2-70 ms, station-dependent)
Dclock The unknown we solve for

Three-Layer Architecture

Layer 1: Single Broadcast
Measures tick rate, but floating in time
Layer 2: Multi-Frequency
Dispersion → TEC → path delay correction
Layer 3: Multi-Station
Geometry lock → integrity validation
💡 The Key Insight

Single-broadcast methods achieve only ±5-10 ms due to mode ambiguity. Our multi-broadcast fusion achieves ±0.5 ms by: (1) Calibrating systematic propagation biases per station, (2) Fusing 9-17 independent measurements with inverse variance weighting, and (3) Validating against physical constraints (cross-station consistency, continuity).
